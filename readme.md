# 🏟️ Арена Моделей

**Arena of Models** — это веб-приложение для проведения интеллектуальных диалогов между двумя языковыми моделями с последующей оценкой независимым судьёй.

## 🚀 Возможности

- **Диалоги моделей** — две LLM ведут структурированный диалог на заданную тему
- **Стриминг в реальном времени** — ответы отображаются по мере генерации (SSE)
- **Независимый судья** — третья модель анализирует диалог и выносит вердикт
- **Сохранение истории** — все диалоги сохраняются в SQLite
- **Просмотр и экспорт** — возможность просмотреть любой диалог и экспортировать в `.txt`
- **Остановка диалога** — возможность прервать диалог в любой момент
- **Автопереподключение** — при обрыве SSE соединение восстанавливается
- **Чистка ответов** — автоматическое удаление префиксов, эмодзи и англицизмов
- **Адаптивный интерфейс** — работает на десктопе и мобильных устройствах

## 📦 Требования

- **Python 3.8+**
- **Ollama** (установлен и запущен локально)
- Минимум **16 ГБ ОЗУ** (для моделей 7B-14B)
- Рекомендуемые модели:
  - `llama3.2:latest` (3B) — быстрая, креативная
  - `ruadapt-qwen2.5-14b:latest` (14B) — сильный русский язык
  - `mistral:7b` (7B) — компромиссный вариант
  - `gemma2:9b` (9B) — альтернатива

## 🔧 Установка

### 1. Клонирование репозитория

```bash
git clone https://github.com/yourusername/arena-of-models.git
cd arena-of-models
```

### 2. Установка зависимостей

```bash
pip install fastapi uvicorn httpx
```

### 3. Установка и запуск Ollama

```bash
# Linux / macOS
curl -fsSL https://ollama.com/install.sh | sh

# Запуск сервера
ollama serve
```

### 4. Загрузка моделей

```bash
ollama pull llama3.2:latest
ollama pull ruadapt-qwen2.5-14b:latest
ollama pull mistral:7b
```

## 🎮 Использование

### Основной экран (Арена)

1. Выберите **Модель А** и **Модель Б** из выпадающих списков
2. Введите **тему диалога** (или оставьте предложенную)
3. Нажмите **«Старт»**
4. Наблюдайте за диалогом в реальном времени
5. По окончании судья вынесет вердикт
6. Кнопка **«Остановить»** прерывает диалог досрочно

### Вкладка «История»

- Просмотр всех сохранённых диалогов
- Клик по сессии открывает модальное окно с полным диалогом
- **Экспорт в .txt** — сохранение диалога в текстовый файл
- **Удалить** — удаление сессии из базы данных

### Горячие клавиши

| Клавиша | Действие |
|---------|----------|
| `Enter` в поле темы | Запуск арены |

## 📡 API Endpoints

| Метод | Endpoint | Описание |
|-------|----------|----------|
| `GET` | `/` | Отдача `index.html` |
| `GET` | `/api/models` | Список доступных моделей Ollama |
| `GET` | `/api/sessions` | Список всех сессий |
| `GET` | `/api/session/{sid}` | Получение диалога по ID |
| `POST` | `/api/arena/start` | Запуск новой арены |
| `GET` | `/api/arena/stream/{sid}` | SSE-поток диалога |
| `POST` | `/api/arena/stop/{sid}` | Остановка активной сессии |
| `DELETE` | `/api/session/{sid}` | Удаление сессии |

### Пример запроса на старт арены

```json
POST /api/arena/start
{
  "topic": "Влияние ИИ на образование",
  "model_a": "llama3.2:latest",
  "model_b": "ruadapt-qwen2.5-14b:latest",
  "max_turns": 8
}
```

### Пример SSE-сообщений

```javascript
// Модель печатает
{"type": "typing", "model": "A"}

// Ответ модели
{"type": "message", "model": "A", "content": "Искусственный интеллект меняет..."}

// Судья думает
{"type": "judge_thinking"}

// Вердикт
{"type": "verdict", "content": "Модель А сильные стороны: ..."}

// Завершение
{"type": "finished"}
```

## ⚙️ Настройка

### Конфигурация в `app.py`

```python
# ========== КОНФИГ ==========
OLLAMA_URL = "http://127.0.0.1:11434/api/chat"
DEFAULT_MODELS = ["llama3.2:latest", "ruadapt-qwen2.5-14b:latest"]
JUDGE_MODEL = "ruadapt-qwen2.5-14b:latest"
TEMPERATURE = 0.8          # Креативность (0.0 - 1.0)
MAX_TURNS = 8              # Максимальное количество ходов
MAX_TOKENS = 400           # Токенов на ответ модели
MAX_TOKENS_JUDGE = 800     # Токенов на вердикт судьи
DB_PATH = "arena.db"       # Путь к базе данных
```

### Изменение промптов

Промпты находятся в функциях:
- `_get_system_prompt()` — системный промпт для моделей
- `_get_judge_prompt()` — промпт для судьи

### Добавление новых моделей

1. Установите модель через Ollama:
   ```bash
   ollama pull gemma2:9b
   ```

2. Модель автоматически появится в выпадающих списках

## 🗄️ Структура базы данных

### Таблица `arena_sessions`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | TEXT | UUID сессии |
| `model_a` | TEXT | Название модели А |
| `model_b` | TEXT | Название модели Б |
| `topic` | TEXT | Тема диалога |
| `started_at` | TIMESTAMP | Время начала |
| `ended_at` | TIMESTAMP | Время завершения |
| `turns_count` | INTEGER | Количество ходов |
| `status` | TEXT | `active` / `finished` / `stopped` |
| `judge_verdict` | TEXT | Вердикт судьи |

### Таблица `arena_messages`

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | INTEGER | Автоинкремент |
| `session_id` | TEXT | Внешний ключ к сессии |
| `model` | TEXT | `A` или `B` |
| `content` | TEXT | Текст сообщения |
| `turn_num` | INTEGER | Номер хода |
| `created_at` | TIMESTAMP | Время создания |

## 🔧 Решение проблем

### ❌ `Connection refused` при старте

**Причина:** Ollama не запущен или работает на другом порту.

**Решение:**
```bash
ollama serve
# или
systemctl start ollama  # Linux
```

### ❌ Модели не отображаются в списке

**Причина:** Ollama запущен, но модели не загружены.

**Решение:**
```bash
ollama list          # Проверить установленные модели
ollama pull llama3.2:latest  # Загрузить модель
```

### ❌ Ответы моделей на английском или с эмодзи

**Решение:** Увеличьте строгость очистки в функции `_clean()` или измените промпт `_get_system_prompt()`.

### ❌ Модели повторяют друг друга

**Решение:** Увеличьте `TEMPERATURE` до `0.9` и добавьте более строгий `anti_repeat` в промпт.

### ❌ Ошибка `NS_ERROR_NET_PARTIAL_TRANSFER` в Firefox

**Причина:** Firefox агрессивно закрывает SSE-соединения.

**Решение:** Включено автоматическое переподключение в `index.html`.
